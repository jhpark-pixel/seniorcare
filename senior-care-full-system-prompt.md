# 시니어 주거시설 통합 관리 시스템 (ERP) — Claude Code 프롬프트

아래 `---` 이후 전체를 Claude Code에 붙여넣으세요.

---

시니어(고령자) 주거시설 통합 관리 ERP 시스템을 풀스택으로 개발해줘.
기존 케어닥(CareDoc) 시스템의 구조를 참고하되, IoT 낙상감지 + AI 건강분석 + 프로그램 추천까지 확장한다.

## 기술 스택

- Frontend: React + TypeScript + Tailwind CSS
- Backend: Node.js + Express + TypeScript
- DB: SQLite (개발) / PostgreSQL (운영)
- ORM: Prisma
- 차트: Recharts
- 실시간: Socket.IO
- PDF: pdfkit
- 인증: JWT
- 한국어 UI, 반응형

**좌측 메뉴:**

```

  └─ 청약자가입현황

📄 계약관리
  ├─ 계약서등록 및 조회
  ├─ 보증금입금 및 조회
  └─ 계약자현황

🏠 입소관리
  ├─ 입소자등록
  ├─ 입퇴소자현황
  └─ 입소자 목록 (마스터 테이블)

※ 입소자 목록 (마스터 테이블) 컬럼 — 스크린샷 정확 반영:
| 연번 | 입소자명 | 성별 | 나이 | 주민등록번호 | 등급 | 최초계약일 | 계약시작일 | 계약종료일 | 계약구분 | 호실 | 유형 | 기준생활비(상급점실료+식사비) | 기준매출액 | 프로모션할인비(월기준) | 기준일단가 | 이용시작일 | 이용종료일 |

💰 청구관리
  ├─ 월청구내역
  ├─ 월청구산출 (일반관리비)

  ├─ 월청구산출 (수도광열비)
  └─ 월청구내역현황

🚗 장기외출
  └─ 장기외출신청등록

🏢 시설자산관리
  ├─ 비품/자산대장
  └─ 비품/자산 유지보수 요청서

🔄 재계약관리
  └─ 재계약대상자조회

🚪 퇴소관리
  └─ 퇴소신청

💳 정산관리
```

### 탭 7. 경영통계 ⚠️ 관리자 전용 (시설장 + 경영관리자만 접근 가능)

이 탭은 로그인한 직원의 role이 DIRECTOR(시설장) 또는 MANAGER(경영관리자)인 경우에만 상단 탭에 표시된다. 다른 직원에게는 탭 자체가 보이지 않는다. 접근 시도 시 "권한이 없습니다" 안내.

```
📈 입소현황 통계
  ├─ 거주자 종합현황 (동별/타입별 세대수, 계약률, 입주율, 공실률)
  ├─ 입퇴소 추이 (월별/분기별/연별 라인차트)
  ├─ 계약/청약 현황 (파이프라인: 상담→청약→계약→입소 전환율)
  └─ 입주자 인구통계 (연령대별, 성별, 장기요양등급별 분포)

💰 재무통계
  ├─ 월별 매출현황 (막대+라인 복합 차트)
  ├─ 항목별 매출 분석 (관리비/식사비/수도광열비/서비스비 비중 파이차트)
  ├─ 미수금 현황 (입주자별 미납 목록, 미납 기간별 분류, 총 미수금액)
  ├─ 보증금 현황 (총 보증금 보유액, 환불 예정액, 월별 입출금)
  ├─ 수납률 추이 (월별 청구 대비 수납 비율)
  └─ 매출 예측 (현재 계약 기반 향후 6개월 예상 매출)

📊 운영통계
  ├─ 서비스 신청 통계 (유형별/월별 추이, 고혈압/당뇨/신장/저작곤란/병원동행/민원)
  ├─ 프로그램 운영 현황 (카테고리별 참여율, 인기 프로그램 TOP 10)
  ├─ 직원 업무량 분석 (담당 입주자 수, 상담 건수, 서비스 처리 건수)
  ├─ 민원 분석 (유형별, 처리 소요시간, 만족도)
  └─ 장기외출/입원 현황 (현재 외출자 수, 평균 외출 기간)

🏥 건강통계
  ├─ 질환별 입주자 분포 (도넛 차트 + 상세 테이블)
  ├─ 건강상담 실적 (당일/당월/반기 — 유형별 건수, 상담사별 실적)
  ├─ 낙상사고 통계 (월별 발생 건수, 장소별/시간대별 분포, 심각도, 재발률)
  ├─ 위험군 현황 (건강점수 60점 미만 입주자, 집중케어 대상자 추이)
  ├─ 투약 현황 (전체 복용약물 종류, 다약제 복용자 현황)
  └─ IoT 알림 분석 (일별 알림 건수, 오탐율, 디바이스 가동률)

📋 입주자 종합 분석
  ├─ 입주자 전체 데이터 조회 (아래 '입주자 종합 프로필 페이지' 참조)
  ├─ 입주자 비교 분석 (2~5명 선택 비교: 건강점수, 서비스이용, 비용)
  ├─ 고위험군 리포트 (건강+재무+사회적 고립 종합 위험도)
  └─ 엑셀 일괄 다운로드 (전체 입주자 데이터 + 필터 조건 적용)

📑 경영 리포트
  ├─ 월간 경영보고서 (PDF 자동 생성: 입주현황+재무+운영+건강 종합)
  ├─ 분기 경영보고서
  ├─ 연간 경영보고서
  └─ 커스텀 리포트 (기간/항목 선택하여 맞춤 보고서 생성)
```

### 탭 8. 경영관리자 ⚠️ 관리자 전용 (시설장만 접근 가능)

이 탭은 role이 DIRECTOR(시설장)인 경우에만 표시. 경영관리자(MANAGER) 포함 다른 모든 역할은 접근 불가.

```
👤 직원관리
  ├─ 직원등록/조회
  ├─ 근무스케줄
  └─ 권한관리 (역할별 메뉴 접근 권한 매트릭스 설정)

⚙️ 시스템설정
  ├─ 기본정보 설정 (시설명, 주소, 대표자, 사업자번호, 로고)
  ├─ 코드관리 (질환코드, 서비스코드, 프로그램카테고리 등)
  ├─ 요금설정 (타입별 기본 관리비/식사비/수도광열비 단가)
  ├─ 알림설정 (낙상/이상바이탈/미수금 알림 임계값)
  └─ 데이터 백업/복원

🔒 접근권한 매트릭스
  (아래 권한관리 상세 섹션 참조)
```

## 상단 공통 UI

- 좌측 상단: 시설 로고 + 시설명
- 상단 탭 메뉴: 현재 선택 탭은 주황색 배경으로 강조
- 우측 상단 날짜 표시: "2026년 03월 30일"
- 우측 상단 기능 버튼: 신규[F2] | 조회[F3] | 저장[F4] | 삭제[F5] | 인쇄[F6] | 엑셀[F7] | 종료[F12]

## AI 건강 분석 엔진 (규칙 기반, API 없이)

### 건강 종합 점수 (100점)

| 항목 | 배점 | 판정 기준 |
|------|------|----------|
| BMI | 15점 | 정상(18.5~24.9): 15점, 주의(17~18.4 or 25~29.9): 10점, 위험(<17 or ≥30): 5점 |
| 혈압 | 20점 | 정상(<120/80): 20점, 주의(120~139/80~89): 13점, 위험(≥140/90): 7점 |
| 혈당(공복) | 20점 | 정상(70~99): 20점, 주의(100~125): 13점, 위험(≥126 or <70): 7점 |
| 콜레스테롤 | 15점 | 정상(<200): 15점, 주의(200~239): 10점, 위험(≥240): 5점 |
| 활동성 | 15점 | Level1: 15점, Level2: 11점, Level3: 7점, Level4: 3점 |
| 수면/수분 | 15점 | 수면 7~9h + 수분 ≥1500ml: 15점, 부분 충족: 10점, 미충족: 5점 |

### 위험 태그 자동 생성
- BMI < 18.5 → "저체중 위험"
- BMI ≥ 30 → "비만 주의"
- 수축기 ≥ 140 or 이완기 ≥ 90 → "고혈압 주의"
- 공복혈당 ≥ 126 → "혈당 위험"
- 수분 < 1000ml → "탈수 주의"
- 낙상이력 있고 운동능력 Level 2 이상 → "낙상 고위험"
- 수면 < 5시간 연속 3일 → "수면장애 주의"
- 체중 30일간 3kg 이상 변화 → "체중 급변 주의"

### 맞춤 가이드 생성

**(a) 식단 가이드** — 질환별 교차 규칙
- 당뇨: 저당, GI지수 낮은 식품, 탄수화물 제한
- 고혈압: 저염(나트륨 2000mg 이하), DASH 식단
- 신장질환: 저단백, 칼륨/인 제한
- 심장질환: 저지방, 저콜레스테롤
- 연하곤란: 음식 형태 조절 (일반→다진→미음)
- 복수 질환 시 교차 적용하여 최종 식단 산출
- 주간 식단표 (월~일 × 아침/점심/저녁/간식) + 영양소 목표

**(b) 운동 가이드** — 능력별 프로그램
- Level 1: 걷기 30분, 밴드운동, 균형훈련, 스트레칭
- Level 2: 보조기구 보행훈련, 앉아서 상체운동, 가벼운 스트레칭
- Level 3: 휠체어 상체운동, 관절가동범위 운동
- Level 4: 침상 스트레칭, 수동관절운동, 체위변환
- 질환 반영: 관절염→수중운동, 심장→고강도 금지, 골다공증→충격운동 금지
- 낙상 고위험 → 균형훈련 강화 + 단독운동 제한

**(c) 생활 습관 가이드**
- 수면, 수분, 인지활동, 사회활동 권장사항
- 질환별 맞춤 팁

**(d) 프로그램 추천** — 시설 내 프로그램 매칭
- 운동능력 등급 ≥ 요구 등급
- 인지능력 등급 ≥ 요구 등급
- 제외 질환 충돌 없음
- 건강 분석 결과 반영 (낙상위험→균형운동, 인지저하→인지훈련, 고립→사회프로그램)
- 추천 점수: 적합도(40%) + 건강필요도(40%) + 정원여유(20%)
- 추천 카드 UI에 "추천 사유" 표시

## 입주자 종합 프로필 페이지

입소자관리 > 입소관리 > 입소자등록에서 이름 클릭 시, 또는 어디서든 입주자 이름 클릭 시 해당 입주자의 **종합 프로필 페이지**로 이동한다.

이 페이지는 한 명의 입주자에 대한 모든 데이터를 탭(서브탭)으로 나눠서 보여준다:

### 서브탭 A: 기본 정보
- 프로필 사진 (원형 썸네일)
- 이름, 주민등록번호 (뒷자리 마스킹), 나이, 성별
- 혈액형, 장기요양등급, 장애유형/등급
- 개인 연락처 (휴대폰, 자택)
- 이전 주소
- 입소일, 퇴소일, 현재 상태, 입소 경로
- 이전 시설 이력
- 비상연락처 목록 (최대 3명 — 이름, 관계, 연락처, 의료결정권)
- 주 보호자 정보 + 방문 빈도
- 가족관계 메모

### 서브탭 B: 계약/재무 정보 (행정+경영관리자+시설장만 열람 가능)
- 호실 정보: 호실번호, 유형 (특실/1인실/2인실), 층수
- 계약 정보:
  | 항목 | 내용 |
  |------|------|
  | 최초 계약일 | 2024-01-31 |
  | 계약 시작일 | 2026-01-31 |
  | 계약 종료일 | 2027-01-30 |
  | 계약 구분 | 연장 |
- 보증금: 금액, 납부액, 상태 (완납/분납/미납)
- 요금 내역표 (스크린샷 컬럼 반영):
  | 항목 | 금액 (원) |
  |------|----------|
  | 기준 생활비 (상급 점실료+식사비) | 3,630,000 |
  | 기준 매출액 | 3,300,000 |
  | 프로모션 할인 (월) | 0 |
  | 기준 일단가 | 117,857 |
  | **실 청구액** | **3,300,000** |
- 이용기간: 시작일 ~ 종료일
- 납부 방법, 은행/계좌 (마스킹), 납부일
- 최근 12개월 납부 이력 테이블 (청구액/프로모션/실청구/납부액/미납액/상태)
- 미납 현황 강조 (미납 시 빨간색)
- 프로모션 할인 적용자는 할인액 노란색 하이라이트

### 서브탭 C: 건강 정보
- 기저질환 목록 (질환명, 진단일, 중증도, 상태)
- 복용 약물 목록 (약물명, 용량, 복용시간, 처방의)
- 알레르기 목록
- 식이제한 목록
- 건강 종합 점수 (게이지 차트) + 항목별 판정
- 최근 30일 바이탈 추이 차트 (혈압, 혈당, 체중, 심박수)
- 위험 태그 배지
- 정기검진 이력 + 다음 예정일

### 서브탭 D: 성격/성향 프로필
- 성격 유형 (태그 뱃지: 외향적, 사교적 등)
- 기질 특성 (텍스트)
- 소통 방식
- 사회 성향
- 취미/관심사 (태그 뱃지)
- 좋아하는 것 / 싫어하는 것
- 생활 패턴 (일과 메모)
- 수면 패턴
- 종교
- 식사 선호
- 동실자 선호도 (2인실인 경우)
- 특이사항/주의사항 (눈에 띄게 노란색 배경 박스)

### 서브탭 E: 서비스/프로그램
- 현재 참여중인 프로그램 목록 + 주간 시간표
- 프로그램 출석률 (%)
- AI 추천 프로그램 카드 (미참여 추천)
- 서비스 신청 이력
- 병원동행 이력
- 민원 이력

### 서브탭 F: 케어 이력
- 집중케어 등록 여부 + 이력
- 낙상 이벤트 이력 (날짜, 장소, 심각도, 대응결과)
- 안전사고 이력
- 건강상담 이력
- IoT 디바이스 연결 상태
- 장기외출 이력

### 서브탭 G: 활동 타임라인
- 모든 이벤트를 시간순 타임라인으로 통합 표시:
  입소 → 건강기록 → 상담 → 프로그램참여 → 낙상 → 서비스신청 → 검진 등
- 필터: 유형별, 기간별
- 무한스크롤 또는 페이지네이션

## 경영통계 대시보드 상세 설계 (관리자 전용)

### 1. 거주자 종합현황 페이지

**상단 KPI 카드 (4개)**
- 총 입주자: 숫자 + 전월 대비 증감
- 계약률: % + 게이지
- 입주율: % + 게이지
- 공실수: 숫자 + 전월 대비

**거주자 현황 테이블** (케어닥 스크린샷의 위젯 6 확장판)
| 동 | 타입 | 총 세대 | 계약 세대 | 계약률 | 입주 세대 | 입주율 | 입주자 수 | 공실 수 |
테이블 하단에 합계 행

**차트 영역**
- 입퇴소 추이 라인차트 (최근 12개월)
- 계약 파이프라인 깔때기: 상담 → 청약 → 계약 → 입소 (각 단계 건수 + 전환율)
- 입주자 인구통계: 연령대 분포 막대, 성별 파이, 장기요양등급 도넛

### 2. 재무통계 페이지

**상단 KPI 카드**
- 이번 달 매출 총액
- 수납률 (%)
- 미수금 총액 (빨간색 강조)
- 보증금 보유 총액

**월별 매출 추이** (막대+라인 복합: 청구액 막대 + 수납액 라인)

**항목별 매출 분석** (파이차트)
- 일반관리비 / 식사비 / 수도광열비 / 서비스비 비중

**미수금 상세 테이블** (관리자만 접근)
| 입주자명 | 호실 | 미납 개월 | 미납 총액 | 최종 납부일 | 연락처 | 보호자 연락처 | 조치상태 |
- 미납 3개월 이상 빨간색, 1~2개월 노란색
- "독촉 알림 발송" 버튼

**보증금 현황 테이블**
| 입주자명 | 보증금액 | 납부일 | 상태 | 예상환불일 |

**매출 예측** (현재 계약 기반)
- 향후 6개월 예상 매출 라인차트
- 만료 예정 계약 건수 표시

### 3. 운영통계 페이지

**서비스 신청 추이** (월별 누적 막대차트 — 유형별 컬러)

**프로그램 참여율** (수평 막대: 프로그램별 정원 대비 참여율)
- TOP 10 인기 프로그램
- 참여율 저조 프로그램 경고

**직원 업무량 분석** (관리자만)
| 직원명 | 역할 | 담당 입주자 | 이번 달 상담 건수 | 서비스 처리 건수 | 평균 처리 시간 |

**민원 분석**
- 유형별 파이차트
- 월별 접수/완료 추이
- 평균 처리 소요일

### 4. 건강통계 페이지

**질환별 분포** (도넛 차트 — 고혈압 N명, 당뇨 N명, 치매 N명 등)

**건강상담 실적 매트릭스** (케어닥 위젯 4 확장)
| 구분 | 건강상담 | 외래 | 정기검진 | 체육운동 | 약역관련 | 합계 |
|------|---------|------|---------|---------|---------|------|
| 당일 | | | | | | |
| 금주 | | | | | | |
| 당월 | | | | | | |
| 분기 | | | | | | |
| 반기 | | | | | | |

**낙상사고 분석**
- 월별 발생 건수 추이
- 장소별 분포 (호실/복도/화장실/식당/야외)
- 시간대별 분포 (0~6시/6~12시/12~18시/18~24시)
- 심각도별 (경상/중상/병원이송)
- 낙상 후 재발률

**위험군 대시보드**
- 건강점수 60점 미만 입주자 목록 + 추이
- 집중케어 대상자 현황 (현재/추이)
- 다약제 복용자 (5종 이상) 목록

### 5. 입주자 전체 데이터 조회 (관리자 전용 마스터 테이블)

전체 입주자의 핵심 데이터를 하나의 대형 테이블로 조회:

| 호실 | 이름 | 나이 | 성별 | 주민번호 | 연락처 | 보호자 | 보호자연락처 | 입소일 | 보증금 | 월생활비 | 납부상태 | 미납액 | 건강점수 | 위험태그 | 운동등급 | 인지등급 | 기저질환 | 성격유형 | 사회성향 | 프로그램수 |

- 모든 컬럼 정렬/필터 가능
- 다중 조건 검색 (예: "건강점수 60점 미만 + 미납 있음 + 낙상 고위험")
- 컬럼 표시/숨김 토글 (사용자가 원하는 컬럼만 선택)
- 선택한 입주자 비교 분석 (체크박스 → "비교" 버튼)
- 엑셀 다운로드 (현재 필터 조건 반영)
- 주민등록번호, 계좌번호 등 민감정보는 권한 체크 후 표시 (마스킹 기본)
- "전체보기" 버튼 클릭 + 비밀번호 재입력 시 마스킹 해제

### 6. 경영 리포트 자동 생성

**월간 경영보고서 PDF** — 자동 생성 또는 수동 트리거
구성:
1. 표지 (시설명, 보고 기간, 작성일)
2. 요약 (핵심 KPI 5개: 입주율, 매출, 수납률, 미수금, 건강점수 평균)
3. 입주 현황 (거주자 현황 테이블 + 입퇴소 추이)
4. 재무 현황 (매출 + 미수금 + 보증금)
5. 건강 현황 (질환 분포 + 위험군 + 낙상 + 상담실적)
6. 운영 현황 (서비스 + 프로그램 + 민원)
7. 특이사항 및 조치 필요 항목
8. 다음 달 계획

**분기/연간 보고서**도 동일 구조 + 기간별 비교 추이 포함

  - 90세 이상(강정순, 구자윤, 박정희, 조용근, 한대현, 허용오, 최종현): 골다공증/관절염 높은 확률
  - 80대: 고혈압, 당뇨 중 1~2개
  - 67세(오윤희): 상대적으로 건강
- 복용 약물 1~4개
- 알레르기 0~2개
- 식이제한: 질환에 맞춰 자동 배정
- 운동능력: 등급외 → Level 1~2, 3등급 → Level 2~3, 4등급 → Level 3
- 인지능력: 대부분 정상, 장기요양 등급자는 경도인지장애 가능

**성격/성향 프로필** (25명 모두 다양하게)
- 각각 다른 성격 유형 조합 배정
- 취미/관심사 1~3개
- 생활패턴 메모
- 특이사항 (해당자만)

**비상연락처**
- 1~2명씩 (이름, 관계, 전화번호, 의료결정권)

**계약/재무**
- 위 25명의 계약 데이터 정확히 반영
- 보증금: 특실 5,000만원, 1인실 4,000만원, 2인실 3,000만원 (기본값)
- 최근 3개월 청구/수납 이력
- 1~2명은 미납 상태로 생성 (경영통계 테스트용)

**기타 시드 데이터**
- 프로그램 12개 (카테고리별 2개)
- 월간행사 3개

- 낙상이벤트 3건 (대응기록 포함)
- 안전사고 2건
- 서비스신청 8건 (유형 다양)

## UI/UX 요구사항

- 레이아웃: 상단 탭바 + 좌측 사이드바(트리형) + 메인 콘텐츠 + 하단 상태바
- 상단 탭: 현재 선택 = 주황색(#E67E22) 배경 + 흰색 글씨
- **상단 탭 표시는 로그인 사용자 권한에 따라 필터링** (경영통계/경영관리자 탭은 해당 권한자에게만 표시)
- 좌측 사이드바: 접힘/펼침 가능, 현재 선택 = 주황색 배경
- 테이블: 줄무늬 배경, 헤더 회색, 정렬/필터/검색 지원
- 위험 표시: 빨강(#E74C3C), 주의: 노랑(#F39C12), 정상: 초록(#27AE60)
- 반응형 (태블릿까지 대응)
- 상단 단축키 버튼 (F2~F12) 동작 구현
- 낙상 감지 실시간 알림: 화면 상단 배너 + 알림음

### 민감정보 보안 규칙
- 주민등록번호: DB에 AES-256 암호화 저장. 화면에는 앞 6자리만 표시 (예: 480315-1******). Level 3 이상 권한자만 "전체보기" 가능 (비밀번호 재확인 후 세션 내 1회 해제, 30초 후 자동 마스킹 복원).
- 계좌번호: 동일하게 암호화 저장 + 마스킹 (예: 국민 ***-****-1234).
- 주민번호/계좌번호 조회 시 ActivityLog에 반드시 기록 (누가 언제 어떤 입주자의 민감정보를 열람했는지).
- 엑셀 다운로드 시 민감정보 컬럼은 마스킹 상태로 내보내기 (원본 내보내기는 시설장만 가능).

## 개발 순서

1. Prisma 스키마 전체 작성 + 마이그레이션 (민감정보 암호화 필드 포함)
2. 시드 데이터 생성 스크립트 (8명 입주자 전체 프로필 + 재무 + 성격)
3. Express API — 입주자 CRUD + 건강기록 CRUD
4. React 레이아웃 (탭 + 사이드바 + 라우팅 + 권한별 메뉴 필터링)
5. 로그인/인증 + 역할 기반 접근제어 (JWT + 권한 매트릭스)
6. 홈 대시보드 (7개 위젯)
7. 입소자관리 전체 메뉴
8. **입주자 종합 프로필 페이지** (7개 서브탭 — 기본/재무/건강/성격/서비스/케어/타임라인)
9. 건강관리 — 바이탈 입력 + 차트
10. 건강분석표 + AI 가이드 생성 엔진
11. IoT 낙상감지 (시뮬레이터 + Socket.IO + 알림)
12. 커뮤니티 — 프로그램 관리 + AI 추천
13. 컨시어지 — 서비스/병원동행/민원
14. 식사관리 — 식단 + 특별식
15. 시설관리 — 호실 + IoT장치
16. 청구/정산 (월 생활비 산출 + 수납관리 + 미수금)
17. **경영통계 대시보드** (관리자 전용 — 6개 서브페이지)
18. **경영 리포트 PDF 자동생성** (월간/분기/연간)
19. 개인별 건강 PDF 리포트
20. 경영관리자 — 직원관리 + 시스템설정 + 권한매트릭스 UI

DB 스키마부터 시작해서 위 순서대로 하나씩 구현해줘.
각 단계마다 동작 확인 가능하도록 만들고, 완료되면 다음 단계로 넘어가줘.
